CONTENT SPECIFICATION — CORE DOMAIN OBJECT
==========================================

Every piece of generated content originates from a ContentSpecification.
No generator may call the LLM without one.


interface ContentSpecification {

  language: string;

  level: string;

  provider?: string;

  contentType: string;

  targetWords?: string[];

  grammarTopics?: string[];

  vocabularyDomains?: string[];

  canDoStatements?: string[];

  constraints?: object;
}


EXTENDED FIELDS (implementation phase 03)
-----------------------------------------

These fields are merged by KnowledgeEngine from CEFR + Language + Provider:

  examStructure?: {
    modules: ModuleDefinition[];
    totalMinutes?: number;
    scoring?: ScoringRules;
  };

  topic?: string;

  skills?: string[];           // reading | listening | writing | speaking | grammar | vocabulary

  outputSchema?: object;       // JSON shape hints for PromptBuilder

  locale?: string;             // BCP-47, e.g. de-DE, en-GB, es-ES

  metadata?: {
    goalId?: string;
    source?: 'ai' | 'pool' | 'library' | 'demo';
    version: string;             // engine version for migration
  };


DOMAIN ENTITIES
---------------

Language
  id: 'german' | 'english' | 'spanish' | …
  code: 'de' | 'en' | 'es' | …
  name: string

CEFRLevel
  id: 'A1' | 'A2' | 'B1' | 'B2' | 'C1' | 'C2'
  descriptorRef: string        // pointer into knowledge/cefr/{level}.json

Provider
  id: 'goethe' | 'cambridge' | 'dele'
  languageIds: string[]         // which languages this provider applies to
  levels: string[]              // supported CEFR levels

ContentType
  id: string                    // see PLAN/07_CONTENT_GENERATORS/CONTENT_TYPES.txt
  requiredSkills: string[]
  supportsChunking: boolean

GrammarTopic
  id: string
  languageId: string
  level: string
  label: string
  description: string

VocabularyDomain
  id: string
  languageId: string
  level: string
  topics: string[]

CanDoStatement
  id: string
  level: string
  skill: string
  statement: string             // CEFR can-do text

AssessmentCriterion
  id: string
  providerId?: string
  skill: string
  bands: object                 // scoring rubric reference


INVARIANTS
----------

1. language + level are always required.
2. provider is required when contentType is Exam or MiniExam.
3. targetWords only from user state — never copied into knowledge base.
4. constraints holds provider-specific limits (word counts, part counts) — not prompts.
