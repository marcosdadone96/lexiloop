PROVIDER ADAPTERS — GOAL
========================

The same ContentSpecification must be able to generate:

  German Goethe B1
  English Cambridge B1
  Spanish DELE B1

without changing the generation pipeline.


WHAT CHANGES PER PROVIDER
-------------------------
Only spec.examStructure and spec.constraints.taskTypes — merged by adapter from knowledge/providers/{id}.json.


WHAT DOES NOT CHANGE
--------------------
KnowledgeEngine.merge()
PromptBuilder.buildPrompt(spec)
ExamGenerator.generate(spec)
LLMAdapter.complete()


ACCEPTANCE TEST
---------------
Given identical contentType=Exam, level=B1, different provider:

  buildSpec({ language, level, provider }) → three specs
  buildPrompt(spec) → three different chunk plans, same function
  output JSON validates against shared exam schema + provider module keys
